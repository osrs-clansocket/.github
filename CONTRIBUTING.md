# Contributing to ClanSocket

Thanks for taking the time to contribute. ClanSocket is an open-source platform
for Old School RuneScape clans, spread across several repositories under the
[osrs-clansocket](https://github.com/osrs-clansocket) org. This guide is the
org-wide default; an individual repo may add its own `CONTRIBUTING.md` that
takes precedence.

## Ways to contribute

- **Report a bug** or **request a feature** — open an issue on the relevant
  repository (use the issue forms).
- **Improve docs** — corrections and clarifications are welcome.
- **Send a pull request** — fixes and features, ideally tied to an open issue.

## Before you start

- Each repository is self-contained. Read its `README.md` first — it states the
  stack, how to build, and any repo-specific conventions.
- For larger changes, open an issue to discuss the approach before writing code.
  It saves everyone a rejected PR.
- Check existing issues and PRs so you're not duplicating work.

## Pull requests

1. Fork the repo and create a branch from the default branch.
2. Keep the change focused — one concern per PR.
3. Make sure the project still builds, lints, and tests pass (see the repo README).
4. Write a clear PR description: what changed, why, and how you tested it. Link
   the issue it closes.
5. Be responsive to review. Maintainers may request changes.

## Licensing

ClanSocket is **BSD-2-Clause**. By contributing, you agree your work is
licensed under it. New source files should carry the standard BSD-2-Clause
copyright header used across the org (the `clansocket-licensing` tooling adds
and verifies these).

All participation is governed by our [Code of Conduct](CODE_OF_CONDUCT.md).
