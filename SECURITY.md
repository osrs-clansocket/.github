# Security Policy

We take the security of ClanSocket seriously — the platform handles live
player telemetry, so responsible disclosure matters.

## Reporting a vulnerability

**Please do not open a public issue for security problems.**

Email **jay@banes-lab.com** with the details below. If Private Vulnerability
Reporting is enabled on the affected repository, you can also use the
**"Report a vulnerability"** button on its **Security** tab.

When reporting, please include:

- the affected repository and surface (dashboard, plugin, server, …);
- a description of the issue and its impact;
- steps to reproduce, or a proof of concept;
- any relevant versions (plugin version, RuneLite version, browser).

## What to expect

- We aim to acknowledge a report within a few days.
- We'll confirm the issue, keep you updated on the fix, and credit you on
  request once it's resolved.
- Please give us reasonable time to release a fix before any public disclosure.

## Scope

In scope: the ClanSocket dashboard and server, the RuneLite plugin, and the
data transport between them. Out of scope: issues in third-party dependencies
(report those upstream), and anything requiring physical access to a user's
machine.
